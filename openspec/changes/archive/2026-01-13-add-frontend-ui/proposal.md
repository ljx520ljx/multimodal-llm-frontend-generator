# Proposal: Phase 3 前端基础 UI

## Summary

实现前端交互原型验证平台，让产品/设计师能够快速体验和验证 UI 交互流程是否合理。

## Motivation

Phase 0-2 已完成后端基础设施和核心服务。现在需要实现前端界面。

**核心定位**：交互原型验证工具，而非代码生成工具。

**主要用户**：产品经理、设计师 —— 需要体验和验证交互流程
**次要用户**：开发者 —— 可查看和导出生成的代码

用户能够：

1. **上传 UI 设计稿** - 拖拽或点击上传多张图片，支持预览和排序
2. **体验交互流程** - 在预览区点击、交互，验证流程是否合理
3. **切换查看模式** - 体验模式（预览为主）和开发模式（含代码编辑）
4. **查看生成代码** - 需要时可展开查看或编辑代码

## Scope

### In Scope

- **双模式布局**
  - **体验模式**（默认）：预览区全屏/大面积展示，专注交互体验
  - **开发模式**：三栏布局（上传 | 代码 | 预览），便于开发调试
  - 一键切换两种模式

- **图片上传组件**
  - 拖拽上传区域
  - 图片预览列表
  - 拖拽排序
  - 删除图片

- **交互预览（核心）**
  - Sandpack 沙箱集成
  - 实时预览 React 代码
  - **支持点击、状态切换等交互**
  - 错误友好提示

- **代码编辑器（可折叠）**
  - Monaco Editor 集成
  - 默认折叠，可展开查看
  - JSX/TSX 语法高亮
  - 支持手动编辑

- **API 集成**
  - 上传图片 API
  - 代码生成 API (SSE)
  - 状态管理 (Zustand)

### Out of Scope

- 对话修改功能 - 留待 Phase 7
- 流程录制/回放功能 - 留待后续版本
- 用户认证
- 历史记录保存
- Vue 框架支持（暂时只支持 React）

## Risks & Mitigations

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Monaco Editor 包体积大 | 首屏加载慢 | 动态导入 + loading 状态 |
| Sandpack 编译错误 | 用户困惑 | 显示友好错误信息 |
| SSE 连接中断 | 生成中断 | 自动重试 + 错误提示 |

## Success Criteria

1. 用户可以拖拽上传多张图片
2. 点击"生成代码"后流式显示生成过程
3. 代码实时在 Sandpack 中预览
4. 用户可以手动编辑代码并实时预览
