## Requirements
### Requirement: Monaco Editor Integration

The system SHALL provide a code editor based on Monaco Editor for viewing and editing generated code.

#### Scenario: Editor displays generated code
- **WHEN** code is generated by the LLM
- **THEN** the Monaco Editor SHALL display the code with syntax highlighting

#### Scenario: Streaming code display
- **WHEN** code is being streamed from the backend
- **THEN** the editor SHALL update in real-time as new code chunks arrive
- **AND** the cursor position SHALL follow the end of the code

### Requirement: JSX/TSX Syntax Support

The system SHALL support JSX/TSX syntax highlighting and language features.

#### Scenario: React code highlighting
- **WHEN** React/JSX code is displayed
- **THEN** JSX tags SHALL be highlighted appropriately
- **AND** TypeScript types SHALL be recognized

### Requirement: Editor Loading State

The system SHALL display a loading skeleton while Monaco Editor is being loaded.

#### Scenario: Initial load
- **WHEN** the page first loads
- **THEN** a skeleton placeholder SHALL be displayed
- **AND** once Monaco is loaded, the actual editor SHALL replace the skeleton

### Requirement: Read-Only Mode During Generation

The system SHALL set the editor to read-only mode while code generation is in progress.

#### Scenario: Generation in progress
- **WHEN** code generation is active
- **THEN** the editor SHALL be in read-only mode
- **AND** user input SHALL be ignored

#### Scenario: Generation completed
- **WHEN** code generation completes
- **THEN** the editor SHALL switch to editable mode
- **AND** user modifications SHALL be allowed

### Requirement: Manual Code Editing

The system SHALL allow users to manually edit the generated code.

#### Scenario: User edits code
- **WHEN** user modifies code in the editor (after generation completes)
- **THEN** the changes SHALL be reflected in the preview
- **AND** the preview SHALL update in real-time

### Requirement: Editor Toolbar

The system SHALL provide a toolbar with editor utility actions.

#### Scenario: Copy code to clipboard
- **GIVEN** code is displayed in the editor
- **WHEN** the user clicks the copy button
- **THEN** the current code SHALL be copied to the clipboard
- **AND** a success indicator SHALL be shown briefly

#### Scenario: Switch between files
- **GIVEN** multiple files exist (App.tsx, styles.css)
- **WHEN** the user clicks a file tab
- **THEN** the editor SHALL display the selected file's content
- **AND** maintain the correct syntax highlighting

### Requirement: Enhanced JSX Support

The system SHALL provide proper JSX/TSX syntax support in the editor.

#### Scenario: JSX tags highlighted
- **WHEN** JSX code is displayed
- **THEN** JSX tags SHALL be highlighted differently from regular elements
- **AND** JSX attributes SHALL be properly colored

#### Scenario: TypeScript types recognized
- **WHEN** TypeScript type annotations are present
- **THEN** type keywords SHALL be highlighted appropriately
- **AND** type errors SHALL be underlined if detected

### Requirement: Multi-file Support

The system SHALL support editing multiple files for the generated component.

#### Scenario: Edit App.tsx
- **GIVEN** the default file is App.tsx
- **WHEN** the user modifies the component code
- **THEN** changes SHALL be reflected in the preview

#### Scenario: Edit styles.css
- **GIVEN** a styles.css file exists
- **WHEN** the user switches to styles.css tab
- **THEN** CSS content SHALL be displayed
- **AND** CSS syntax highlighting SHALL be active
