# 质量保障 Skill

> 版本: 1.0.0 | 最后优化: 初始版本

## 触发条件

当用户完成以下操作时自动激活：
- 完成功能开发，准备提交
- 请求代码审查
- 需要评估变更影响

## 能力定义

### 1. 代码自审
开发完成后自动进行代码审查：
- **规范检查**: ESLint/golint 规则遵循度
- **类型安全**: TypeScript 严格模式检查
- **安全审计**: OWASP Top 10 漏洞扫描
- **性能分析**: 识别潜在性能问题

### 2. 本地自测
自动运行相关测试：
- 执行单元测试
- 运行集成测试
- 检查测试覆盖率
- 验证功能正确性

### 3. 变更影响分析
评估改动对现有功能的影响：
- 依赖链分析
- 接口兼容性检查
- 回归风险评估

## 执行流程

```
1. 代码变更检测
   ├── git diff 分析
   ├── 识别变更文件
   └── 确定变更范围

2. 自动审查
   ├── 静态分析
   ├── 规范检查
   └── 安全扫描

3. 测试执行
   ├── 运行相关测试
   ├── 收集覆盖率
   └── 生成测试报告

4. 影响分析
   ├── 构建依赖图
   ├── 标记受影响模块
   └── 风险评级
```

## 检查清单

### 代码规范
- [ ] 命名规范（camelCase/snake_case）
- [ ] 文件组织结构
- [ ] 注释完整性（仅限复杂逻辑）
- [ ] 导入排序

### 安全检查
- [ ] 无硬编码密钥
- [ ] 输入验证
- [ ] SQL 注入防护
- [ ] XSS 防护

### 性能检查
- [ ] 无 N+1 查询
- [ ] 合理的缓存策略
- [ ] 无内存泄漏风险
