# 项目理解 Skill

> 版本: 1.0.0 | 最后优化: 初始版本

## 触发条件

当用户需要以下场景时自动激活：
- 首次接触项目，需要快速了解
- 理解新需求或 Issue
- 分析代码变更的上下文

## 能力定义

### 1. 快速上手
自动扫描并理解项目：
- 识别项目结构（monorepo / 单仓库）
- 检测技术栈（package.json, go.mod, requirements.txt 等）
- 分析架构模式（MVC, Clean Architecture, 分层架构等）
- 识别关键入口点和核心模块

### 2. 需求理解
从多种来源提取结构化需求：
- Issue 描述 → 功能点拆解
- PRD 文档 → 用户故事提取
- 对话内容 → 意图识别和需求澄清

### 3. 上下文构建
关联项目中的相关信息：
- 代码：找到相关的模块、函数、类型定义
- 文档：关联 README、API 文档、设计文档
- 历史：git log 分析，理解演进脉络

## 执行流程

```
1. 扫描项目根目录
   ├── 读取 package.json / go.mod / pyproject.toml
   ├── 分析目录结构
   └── 识别配置文件

2. 构建项目画像
   ├── 技术栈清单
   ├── 架构图谱
   └── 核心模块列表

3. 建立上下文索引
   ├── 入口点映射
   ├── 依赖关系图
   └── 文档关联
```

## 输出格式

```markdown
## 项目概览
- **名称**: {project_name}
- **类型**: {Web应用/CLI工具/库/服务}
- **技术栈**: {主要技术}

## 架构分析
- **模式**: {架构模式}
- **核心模块**: {模块列表}

## 关键文件
- 入口: {entry_points}
- 配置: {config_files}
- 核心逻辑: {core_files}

## 上下文关联
- 相关代码: {related_code}
- 相关文档: {related_docs}
```
