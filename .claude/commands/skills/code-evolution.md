# 代码演进 Skill

> 版本: 1.0.0 | 最后优化: 初始版本

## 触发条件

当用户需要以下场景时激活：
- 识别和处理技术债务
- 执行代码重构
- 迁移到新的技术模式

## 能力定义

### 1. 重构建议
识别技术债务并提出方案：
- **代码异味检测**: 重复代码、过长函数、过大类
- **复杂度分析**: 圈复杂度、认知复杂度
- **依赖分析**: 循环依赖、过度耦合
- **优化建议**: 基于最佳实践的改进方案

### 2. 渐进式重构
小步安全地改进代码结构：
- 保持每次变更最小化
- 确保每步可独立验证
- 维护向后兼容性
- 提供回滚策略

### 3. 模式迁移
将旧模式迁移到新的最佳实践：
- Class Component → Functional Component + Hooks
- Callback → Promise → Async/Await
- REST → GraphQL（如适用）
- 旧状态管理 → 现代方案

## 执行流程

```
1. 现状分析
   ├── 代码质量扫描
   ├── 技术债务识别
   └── 优先级排序

2. 方案设计
   ├── 目标架构定义
   ├── 迁移路径规划
   └── 风险评估

3. 渐进实施
   ├── 第一步：最小可行改动
   ├── 验证：测试通过
   ├── 下一步：增量改进
   └── 循环直到完成

4. 验证确认
   ├── 功能回归测试
   ├── 性能对比
   └── 代码质量评分
```

## 重构模式库

### 常见重构操作

| 模式 | 触发条件 | 改进方案 |
|------|----------|----------|
| 提取函数 | 函数过长 (>50行) | 按职责拆分 |
| 提取组件 | JSX 嵌套过深 | 拆分为子组件 |
| 引入接口 | 硬编码依赖 | 依赖注入 |
| 合并重复 | 相似代码块 | 提取公共函数 |
| 简化条件 | 复杂条件判断 | 策略模式/早返回 |
